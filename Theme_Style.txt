Theme
======

- Цвета
- Стили для всех типов дочерних View
  (например, materialButtonStyle, listViewStyle, textInputStyle, chipStyle, ...)
- Настройки (например цвет системного ststusBar'а)

Стиль является ТЕМОЙ, если наследуется от дефолтовой темы. И вообще тема может наследоваться только от темы.

Обрати внимание:
===============
ТЕМА наследуется от Theme.xxx.xxx...
СТИЛЬ наследуется от Widget.xxx.xxx...

Тема содержит ПАРЫ имя:ЗНАЧЕНИЕ. Стиль может назначать ЗНАЧЕНИЕ атрибуту View ссылаясь на ИМЯ этого значениев теме.

Атрибуты ТЕМЫ - это не атрибуты стиля. Атрибуты темы - это именованные значения, а атрибуты стиля ссылаются
на эти имена, чтобы получить через них значение для себя. Итак, атрибуты темы есть именованные значения для 
атрибутов стилей.

ThemeOverlay - это темы для переопределения атрибутов основной темы или другой "вышестоящей" overlay-темы.

MainTheme -> Activity
  ThemeOverlay1 -> FrameLayout
    ThemeOverlay2 -> Button
	
Например, ThemeOverlay2 может переопределить какой-то атрибут, который ранее переопределила ThemeOverlay1.
Переопределить атрибут темы - значит присвоитиь ему новое значение.

Атрибуты темы нельзя просто взять и переопределить непосредственно внутри макета в свойствах View.
Это, во-первых неправильно, во-вторых это не будет прочитано inflater'ом.

И ещё. Итоговую "склейку" значений атрибутов темы делает ContextThemeWrapper.
